# Telegram response reminder bot

Бот для рабочих групповых чатов: если в сообщении упомянули сотрудника через `@username`, бот ждёт ответ от этого сотрудника в том же чате. Если ответа нет, бот напоминает в группе каждые 15 минут в рабочее время и один раз пробует отправить личное сообщение через 60 минут.

Бот не привязан к одному `chat_id`: он работает в любом групповом чате или супергруппе, куда его добавили, а ожидания хранит отдельно для каждого чата.

## Поведение

- Ожидания хранятся отдельно для каждого исходного сообщения. Если одного и того же сотрудника упомянули в разных сообщениях, бот контролирует каждое обращение отдельно.
- Дубль одного и того же исходного сообщения не плодится: для пары `chat_id + сотрудник + source_message_id` остаётся одно активное ожидание.
- Если сотрудник упомянул сам себя, ожидание для него не создаётся.
- Бот распознаёт обычные `@username`, кликабельные Telegram-упоминания пользователя без видимого username, а также старается сопоставить видимое имя с уже известными пользователями или активными адресатами в чате.
- Если в одном сообщении упомянуто несколько сотрудников, создаётся отдельное ожидание для каждого, а напоминание перечисляет всех адресатов, кто ещё не ответил.
- Если одно обращение адресовано нескольким сотрудникам, ответ или реакция любого из этих сотрудников закрывает всё обращение целиком.
- Если у сотрудника одно активное обращение, содержательное сообщение в том же чате закрывает его даже без reply.
- Если у сотрудника несколько активных обращений, бот закрывает только явно подходящее обращение: reply на исходник, совпадение по смысловым словам/номерам или реакция на исходник. Если связь неочевидна, бот спрашивает кнопками, считать ли сообщение ответом.
- Если кто-то отвечает reply-ом на исходное контролируемое сообщение, бот закрывает всё обращение по этому исходнику. Это помогает в случаях, когда Telegram показал адресата кликабельным именем, но не передал его как обычный `@username`.
- Реакция адресата на исходное сообщение или на напоминание бота тоже закрывает обращение. Это работает и для сотрудников без `@username`, если бот смог сопоставить их по имени или `user_id`.
- Короткие или спорные ответы вроде `ок`, `ща`, `позже`, `посмотрю`, `уточню` не закрывают ожидание сразу: бот спрашивает кнопками, считать ли это ответом.
- Если адресат ожидания пишет сообщение с новым `@username`, бот считает это переадресацией: закрывает ожидание по первому сотруднику и начинает контролировать нового.
- На втором групповом напоминании бот явно пишет: если ответа не будет в течение интервала напоминаний, руководителю будет отправлен запрос на штраф.
- На третьем групповом напоминании бот пишет, что запрос на решение по штрафу отправлен руководителю в личку.
- В личке руководителю приходят кнопки `Выписать штраф 500 ₽` и `Не штрафовать`. После решения бот редактирует третье напоминание в рабочей группе и фиксирует, назначен штраф или нет. Общая сумма штрафов в рабочий чат не публикуется.
- В напоминаниях есть кнопки `Закрыть`, `Отложить` и, если доступна ссылка, `Открыть сообщение`.
- Если Telegram даёт ссылку на исходное сообщение, бот вшивает её в текст цитаты: по цитате можно нажать и перейти к сообщению.
- Новые групповые напоминания отправляются reply на исходное сообщение, чтобы в обычных группах без deep-link оставался нативный переход через плашку ответа.
- Кнопки может нажать любой участник чата; бот фиксирует в тексте, кто именно закрыл или подтвердил обращение.
- Если личное сообщение сотруднику отправить нельзя, бот пишет служебное уведомление руководителю, если руководитель активировал личку с ботом.
- Если личка сотруднику недоступна, бот делает максимум 3 групповых напоминания по этому ожиданию. Третье напоминание становится финальным и мягко фиксирует, что игнорировать обращения коллег в рабочих чатах недопустимо.
- Каждый день в `09:30` бот отправляет в каждый чат отчёт за прошедший день: кто был упомянут и не ответил по обращениям, созданным вчера. Если за вчера были обращения, но все закрыты, бот пишет, что незакрытых нет.
- В это же время бот дополнительно отправляет руководителю общий личный отчёт по всем чатам за прошедший день. Для этого руководитель должен заранее нажать `/start` в личке с ботом.
- Бот пишет базовые метрики в SQLite и показывает их руководителю командой `/stats`.
- Состояние хранится в SQLite, поэтому активные ожидания сохраняются после перезапуска.
- После запуска бот ждёт `SCHEDULER_STARTUP_GRACE_SECONDS`, прежде чем отправлять просроченные напоминания. Это даёт Telegram время прислать ответы, накопившиеся пока бот перезапускался.

## Команды

Доступны всем участникам:

- `/start` — активировать личные сообщения с ботом.
- `/help` — показать список команд.
- `/chatid` — показать ID текущего чата.

В рабочем чате можно ответить на сообщение-напоминание бота текстом:

- `закрыть` — закрыть обращение.
- `отложить` — перенести следующее напоминание на интервал напоминаний.

Команды руководителя:

- `/stats` — базовая статистика работы бота.
- `/settings` — проверить реальные настройки, которые бот подхватил на сервере.
- `/fines` — подробный отчёт по штрафам за текущий месяц: общая сумма, кто получил штраф, когда, в каком чате и за какое обращение.
- `/fines 2026-05` — такой же отчёт за выбранный месяц.

`/fines` также отправляет CSV-файл с полной детализацией, который можно открыть в Excel или Google Sheets.

## Рабочее время

По умолчанию напоминания отправляются ежедневно с `09:30` до `19:00` по Москве.

Если срок отправки наступил вне рабочего времени, бот переносит отправку на ближайшее рабочее окно. Например, если сотрудника упомянули в `18:50`, первое групповое напоминание будет не в `19:10`, а на следующий день в `09:30`.

## Форматы сообщений

Первое групповое напоминание:

```text
@username[, @username2], нужен ответ по сообщению: <ссылка или короткая цитата>
```

Второе групповое напоминание:

```text
@username[, @username2], второе напоминание: нужен ответ по сообщению: <ссылка или короткая цитата>
Если ответа не будет в течение 15 минут, я отправлю руководителю запрос на штраф 500 ₽.
```

Третье групповое напоминание:

```text
@username[, @username2], третье напоминание по сообщению: <ссылка или короткая цитата>
Запрос на решение по штрафу отправлен руководителю @Fedos_AV в личку.
```

Личный запрос руководителю:

```text
Нужно решение по штрафу.
Чат: <название чата>
Кто не отвечает: @username[, @username2]
Сумма штрафа: 500 ₽
Исходное сообщение: <ссылка или короткая цитата>
```

После решения руководителя в рабочем чате, если штраф назначен:

```text
Штраф назначен: 500 ₽.
Решение принял: @Fedos_AV
Время решения: <дата и время>
Адресаты: @username[, @username2]
Исходное сообщение: <ссылка или короткая цитата>
```

Если штраф не назначен:

```text
Штраф не назначен.
Решение принял: @Fedos_AV
Время решения: <дата и время>
Адресаты: @username[, @username2]
Исходное сообщение: <ссылка или короткая цитата>
```

Общая сумма штрафов не пишется в рабочие чаты. Руководитель смотрит её командой `/fines`; там же есть детализация: кто получил штраф, когда, в каком чате и за какое исходное обращение.

Личное сообщение сотруднику после 60 минут без ответа:

```text
В рабочем чате ждут ваш ответ: <ссылка на сообщение>
```

Для публичных чатов бот использует ссылки вида `https://t.me/<chat_username>/<message_id>`. Для супергрупп без публичного username используется формат `https://t.me/c/<internal_chat_id>/<message_id>`. В обычных группах Telegram может не дать deep-link на сообщение; в этом случае бот оставляет короткую цитату и отправляет напоминание reply на исходное сообщение.

## Ограничения Telegram Bot API

Важно настроить Telegram правильно, иначе бот физически не сможет выполнить часть логики.

- В группах бот не видит все обычные сообщения, если включён privacy mode. По документации Telegram privacy mode включён по умолчанию для ботов, не добавленных администраторами, и ограничивает поток групповых сообщений.
- Чтобы отслеживать обычные сообщения и ответы сотрудников, отключите privacy mode через BotFather.
- Чтобы Telegram присылал боту реакции на сообщения, бот должен быть администратором в группе. В коде явно включён тип обновлений `message_reaction`, но без прав администратора Telegram такие события не отдаёт.
- Бот не может первым начать личный диалог с пользователем. Пользователь должен открыть бота и отправить `/start`.
- Если бот знает только `@username`, но ещё не знает Telegram `user_id`, личное сообщение невозможно. `user_id` появляется, когда пользователь написал боту в личку или когда бот увидел его сообщение в группе.
- Если пользователь не активировал личку с ботом, Telegram вернёт ошибку вроде `Forbidden: bot can't initiate conversation with a user`. Код обрабатывает это и продолжает групповые напоминания.

Официальные разделы Telegram: [Bot Features](https://core.telegram.org/bots/features) и [Bots FAQ / Introduction](https://core.telegram.org/bots).

## Создание бота через BotFather

1. Откройте Telegram и найдите `@BotFather`.
2. Отправьте `/newbot`.
3. Задайте имя и username бота.
4. Скопируйте токен и положите его в `.env` как `BOT_TOKEN`.

## Отключение privacy mode

1. Откройте `@BotFather`.
2. Отправьте `/mybots`.
3. Выберите вашего бота.
4. Откройте `Bot Settings`.
5. Откройте `Group Privacy`.
6. Нажмите `Turn off`.

После изменения privacy mode лучше удалить бота из группы и добавить заново.

## Добавление в групповой чат

1. Добавьте бота в рабочую группу.
2. Для стабильной работы сделайте бота администратором или убедитесь, что privacy mode выключен.
3. Боту нужны права отправлять сообщения в чат. Специальные права на удаление/бан пользователей не нужны.

## Активация личных сообщений сотрудниками

Каждый сотрудник, которому нужны личные напоминания, должен один раз:

1. Открыть диалог с ботом.
2. Нажать `Start` или отправить `/start`.

После этого бот сохранит `user_id` и сможет отправить личное напоминание, если сотрудника ждут в группе.

Руководителю тоже нужно отправить `/start` боту в личку, иначе бот не сможет присылать служебные уведомления об ошибках личных сообщений.

## Команды

- `/start` в личке: активировать личные сообщения.
- `/chatid` в группе или личке: показать ID текущего чата. Для работы бота этот ID не нужно прописывать в `.env`; команда нужна для отладки.
- `/stats`: показать метрики. Доступно только руководителю из `LEADER_USERNAME`.

## Конфигурация

Скопируйте пример:

```bash
cp .env.example .env
```

Переменные:

```env
BOT_TOKEN=123456:replace_me
TIMEZONE=Europe/Moscow
WORKDAY_START=09:30
WORKDAY_END=19:00
DAILY_REPORT_TIME=09:30
REMINDER_INTERVAL_MINUTES=15
DIRECT_MESSAGE_AFTER_MINUTES=60
LEADER_USERNAME=Fedos_AV
ESCALATE_AFTER_REMINDERS=3
MAX_GROUP_REMINDERS_IF_DM_UNREACHABLE=3
FINE_AMOUNT_RUBLES=500
DATABASE_PATH=bot.sqlite3
SCHEDULER_STARTUP_GRACE_SECONDS=45
```

## Локальный запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python -m bot.main
```

## Запуск на сервере

Минимальный вариант:

```bash
git clone <repo-url>
cd <repo>
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
nano .env
python -m bot.main
```

Для постоянной работы используйте `systemd`, Docker или process manager. Пример `systemd` unit:

```ini
[Unit]
Description=Telegram response reminder bot
After=network-online.target

[Service]
WorkingDirectory=/opt/telegram-response-reminder
EnvironmentFile=/opt/telegram-response-reminder/.env
ExecStart=/opt/telegram-response-reminder/.venv/bin/python -m bot.main
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```
